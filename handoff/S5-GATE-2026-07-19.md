## Defects found (queue-ready)

- **D-11 (Lane B, P2): doctor's yt-dlp check and study's ingest
  disagree about what "installed" means.** `doctor.py:168-169` accepts
  binary OR importable module (and reports ok via the module), but
  `study/ingest.py:159` invokes the literal `yt-dlp` binary — an
  environment with the module importable but `Scripts` not on PATH
  (e.g. venv invoked by absolute path, some launcher setups) gets
  "Verdict: fully ready" and then every study fails with "'yt-dlp' was
  not found on PATH". Repro: `<venv>/Scripts/zing.exe doctor` (ok) then
  `<venv>/Scripts/zing.exe study <url>` (fail) from a shell without the
  venv activated. Fix shape: ingest should fall back to
  `sys.executable -m yt_dlp` (doctor already models this), or doctor
  must probe exactly what ingest will run.
- **D-12 (Lane A, P3): all-refs-failed pack build discards the
  per-reference causes.** `profile/packs.py:123` raises "no reference
  could be studied — a preset built from nothing would be a lie"
  without the `failed` list it just collected — when everything fails
  (one shared root cause, like D-11), the one actionable detail
  vanishes. Append the first cause(s) to the PackError text.
- **D-13 (Lane B, P3): doctor still prescribes `--js-runtimes node`
  when the user's yt-dlp config already contains it** (this box:
  `%APPDATA%/yt-dlp/config` has the line; doctor's fix-hint prints it
  anyway). The #201-P1 fix added solver detection; the
  config-already-applied half of Lane C's original #170 finding is
  still open.
- **O-3 (Lane A, observation):** on sources with no burned speech
  captions, `_caption_style` measures style from whatever OCR events
  exist — here 13 sparse overlay/junk events (majority position
  "center" includes 4 junk glyphs) styled all 47 derived captions.
  D-8's rule behaved as specified and the result is defensible
  (center/caps word-pop), but "style measured from n=13 non-speech
  events" deserves a warning the way thin profile stats get one.
- **(evidence, not new)** The gate video reproduces the P-C2 junk-OCR
  class on a fresh video: 6 high-confidence junk glyphs incl. CJK
  fabrications at 0.76–0.98 conf. Filed as additional evidence toward
  P-C2's calibration, which already owns this.
