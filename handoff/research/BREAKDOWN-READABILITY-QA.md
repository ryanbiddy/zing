# Readability & UX Audit: breakdown.md Report (D-Q10)

This document audits the human-readable edit breakdown (`breakdown.md` / `render_markdown`) generated by the study pipeline, analyzing its clarity from the perspective of an everyday content creator.

---

## 1. Technical Jargon & Pipeline Exposure

Content creators are visual storytelling professionals, not data scientists or software engineers. Several metrics and logs currently expose raw internal engine metrics instead of action-oriented insights:

| Jargon Term / Format | Where It Appears | Creator Confusion / Problem | Recommended Creator-Friendly Alternative |
|---|---|---|---|
| **`conf: 0.93`** | `## Captions` table | Raw ML confidence float (0.0 to 1.0). Confusing to read and lacks actionable meaning. | Convert to a percentage or qualitative flag: `93%` or `High` / `Low` confidence. |
| **`confidence 0.70`** | `## Audio` (music bed) | Raw classifier confidence score. | Express as percentage: `70% probability` or simple `Detected (70%)`. |
| **`RMS dBFS`** | `## Audio` (loudness) | Technical decibels relative to full scale. Most creators think in **LUFS** (Loudness Units Full Scale) because platforms normalize based on it. | Reference **LUFS** targets (e.g. -14 LUFS) and explain if the level is within safe platform bounds. |
| **`1s buckets`** | `## Audio` (loudness) | Pipeline implementation detail that is noisy and irrelevant to creators. | Omit the bucket size; just report "Loudness Range" and "Peak Level". |
| **`caption OCR sampled at 8 fps`** | `## Measurement notes` | Engineering log about sampling rate. | Translate to quality warning: `Fast caption detection active (may miss rapid-flashing words under 125ms)`. |
| **`scenedetect not installed`** | `## Measurement notes` | System dependency trace. | Provide actionable solution: `Shot detection inactive. Install scene detection dependencies (run 'zing doctor') to enable.` |

## 2. Visual & Layout Usability Gaps

### A. Raw Keyframe Paths vs. Visual Images
In the `## Shots` table, the keyframe column prints a raw local path:
`| 0 | 0.00 | 1.40 | 1.40s | frames/shot_000.jpg |`
*   **The Issue:** A text path is useless to a creator reading the markdown report. They cannot see the shot.
*   **The Fix:** Embed the keyframes directly in the markdown using markdown image tags, or link them:
    `| 0 | 0.00 | 1.40 | 1.40s | ![Shot 0](file:///E:/AI/projects/zing-lanes/lane-d/tools/eval/real_videos/raw-editing-practice/shot_000.jpg) |`

### B. Unclear Timeline Distribution Formatting
In the `## Pacing` section, the cut density is represented by space-separated integers:
`cuts per 10s window: 2 0 0`
*   **The Issue:** This format is hard to scan. A creator doesn't instantly know if the first `2` represents the start or end of the video, especially on longer clips.
*   **The Fix:** Use labeled intervals:
    *   `0s - 10s: 2 cuts`
    *   `10s - 20s: 0 cuts`
    *   `20s - 30s: 0 cuts`
    Alternatively, represent it as a sparkline or simple bar chart.

### C. Technical Shot Indexing
Shots are listed starting at index `0` (e.g., `#0 (0.00-1.40s)`).
*   **The Issue:** Creators and standard editing software (Premiere, FCP, Resolve) are 1-indexed. Starting at shot `0` feels unnatural.
*   **The Fix:** Convert shot tables and indices to 1-based indexing for the human-readable markdown output.

### D. Caption Position Labels
The position column prints labels like `pos: lower` or `pos: upper`.
*   **The Issue:** "lower" and "upper" are imprecise. Creators think in safe zones or specific video coordinates (e.g., "bottom safe zone", "top-third overlay").
*   **The Fix:** Clarify the position labels relative to standard 9:16 safe boxes (e.g., `Bottom (Safe Zone)`, `Middle (Obstructed by Feed)`, `Top`).
