# SG-4 scan: MCP specification currency vs zing's server

Scope: our MCP server is Lane B's flagship surface and had never been
scanned against the SPEC (only against clients and packaging). Timely:
the 2026-07-28 revision goes final in ~9 days. Live-verified.

## What's coming (2026-07-28, RC locked 2026-05-21)

The largest revision since launch, aimed at a stateless core that
scales on ordinary HTTP infrastructure. Breaking, protocol-wide:

- **initialize/initialized handshake REMOVED** — client info and
  capabilities ride in `_meta` on every request.
- **Protocol-level sessions removed** — servers needing continuity
  "mint an explicit handle from a tool and have the model pass it back
  as an ordinary argument."
- **Roots, Sampling, Logging deprecated** (use tool params/config,
  direct LLM APIs, stderr or OpenTelemetry).
- Missing-resource error: `-32002` -> `-32602 Invalid Params`.
- Tool input/output schemas lift to full JSON Schema 2020-12.
- **stdio transport survives.**

## Zing's exposure: LOW, and measured rather than assumed

| Change | Our state |
|---|---|
| Handshake removal | Lands in the SDK layer, not our handlers; we build on FastMCP |
| Session removal | We hold NO protocol session state — study/render handles are slugs the model passes back, which is precisely the pattern the RC prescribes (B#2's job design, arrived at for Claude Desktop's 60s cap, happens to be spec-forward) |
| Roots / Sampling / Logging | Unused |
| Error code change | We serve tools + prompts, no resources |
| Schema lift | Additive; our JSON-envelope tools are unaffected |
| stdio | Our only transport, and it survives |

## Verified live this cycle (not inferred)

Our installed SDK (mcp 1.28.1) reports `LATEST_PROTOCOL_VERSION =
2025-11-25`. Negotiation probed against the real server binary:

| Client asks | Server answers | Verdict |
|---|---|---|
| `2024-11-05` (our old test pin) | `2024-11-05` | backward compat holds |
| `2025-11-25` (current) | `2025-11-25` | matched |
| `2026-07-28` (future) | `2025-11-25` | correct: answers with its own latest, never echoes an unsupported version, never fails the handshake |

The stdio gate previously exercised only the oldest path; all three
are now parametrized and pinned, so an SDK bump cannot silently change
negotiation behavior.
