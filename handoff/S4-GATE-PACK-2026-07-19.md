   55.074s exact, 1920x1080@30 h264+aac matching frozen meta, and
   frames at 1s and 46s visually matched against the fixture's

measured meta; honest warning emitted: *"study ran without raw mode, so
chosen spans could not be cross-checked against measured keepers"* —

**VO (honest skip)** — `zing render … --voiceover-script vo-line.txt`
(shot prompt 1's line) failed loud, exit 1: *"Kokoro model assets are

`kokoro-onnx` is not installed and cannot be (repo venv is Python

**ffprobe (rendered draft)** — duration 16.700000 exactly (= 13.4 +
3.3); h264 1920x1080 yuv420p 30/1 + aac 48000 Hz stereo. Matches the
EDL.

**Captions (renderer's own probe approach)** — `captions.ass` from
`--keep-work`: 64 word-pop Dialogue events, Karaoke style, `\pos(960,810)`
= `caption_anchor("lower", 1920, 1080)`. Pixel probe per the
integration-test oracle (gray region 400x120 at the anchor, caption-on
t=1.0 vs caption-off t=13.8 inside the computed 13.32–14.28s gap): abs
gray diff **5,574,374** vs the test's >5,000 bar. Visually confirmed in
extracted frames ("ANYWHERE" word-pop at 0.6s; yellow karaoke "TO" at
14.8s).

**OTIO** — `draft.otio` parses with opentimelineio 0.18.1: Timeline
"draft", Video track with 2 clips (frames 0–402 = 0–13.4s; 1350–1449 =
45.0–48.3s @30fps), mirrored Source Audio track, absolute file-URL media
references, JSON `OTIO_SCHEMA: Timeline.1`. Correct mapping of both
keepers.

**Audio — the gate's real find.** Windowed volumedetect on the draft:
0–13.4s speech at −21 dB mean… then **13.4–16.7s at −91 dB: the payoff
line is silent.** And 0–3.3s is anomalously hot (−16.5 dB vs −24 dB
source): keeper 2's speech is **mixed under the opening** instead of
placed at 13.4s. Isolated stepwise with ffmpeg (all repro commands in
the gate scratchpad):

- source media HAS speech at 45–48.3s (−17.8 dB) — not a media problem;
- the exact saved filtergraph reproduces the fault; clip-audio chain
  alone (`…adelay=13400:all=1` → WAV) yields a **3.34s** file — adelay's
  inserted silence never survives;
- `ashowinfo` after adelay: first frame `pts_time:-192153584101141` —
  the inserted silence carries garbage negative PTS on ffmpeg 8.1.1, so
  the chain's trailing `atrim=duration=16.7` (and amix's PTS alignment)
  discards it;
- **verified one-filter fix**: `asetpts=PTS-STARTPTS` immediately after
  `adelay` → silence 0–13.4s, speech lands 13.4–16.7s, 16.7s total.
- All 6 `test_render_integration.py` tests **pass** on this same
  ffmpeg — no test asserts per-clip audio placement (defect D-2).

**Watchable?** — **No.** What a viewer sees: a 16.7s vertical-ish
landscape draft that opens on the hook ("I don't go anywhere without a
watch") with clean word-pop captions, hard-cuts at 13.4s to the watch
close-up and the payoff beat — visually coherent, captions land. What a
viewer *hears*: two takes talking over each other for the first 3.3
seconds, then the payoff line playing **mute** while its captions pop.
Also visible: the source's own burned-in captions under Zing's (the
F-16 stand-in is an edited showcase; expected). This is not a draft you
could hand a creator.

## Honest limits of this gate pack

1. **No VO was rendered anywhere** — Kokoro model absent, runtime
   uninstallable on the venv's Python 3.14. Track 1's "with VO" clause
   is unexercised. Nothing was faked to cover it.
2. **Gate 1's media is a re-fetch**, not the frozen fixture's bytes
   (sha256s differ; 1080p vs the fixture's smaller format). Alignment
   was verified by duration/streams/two-frame visual match, not by hash.
3. **The direction was reused from the S3 run**, not re-judged; all S3
   caveats (F-16 edited stand-in, no raw-mode keepers, loose gate
   profile) carry forward into this draft.
4. **Gate 2's fresh-fetch half is unproven**: YouTube bot-gating blocked
   every fetch during the gate window. The cold-network onboarding claim
   ("fresh machine, seven URLs, one command") remains untested until the
   network path clears (or cookies/PO-token support is a sanctioned
   decision — it was not mine to take).
5. **One full compare, not two** — the personal-profile compare was
   readiness-verified only.
6. **All profiles are measured-only** (every source unjudged) — the
   judged half of profile-grounded comparison is still exercised as
   "honestly empty", same as S2.
7. Caption CPS warnings in Gate 1 are artifacts of my 4-word glue
   chunking; renderer guidance flagged them correctly.

## Defects found (queue-ready)

- **D-1 (Lane C, P1):** render graph — `adelay`'s inserted silence
  carries garbage negative PTS on ffmpeg 8.1.1, so the clip-audio
  chain's trailing `atrim=duration=` drops it: every clip with
  `timeline_start>0` mixes its audio at t=0 and the output tail goes
  silent. Repro: any 2-clip EDL + windowed volumedetect. Verified fix:
  `asetpts=PTS-STARTPTS` immediately after `adelay` in graph.py.
- **D-2 (Lane C):** all render integration tests pass while D-1 ships —
  no test asserts per-clip audio placement; add windowed level probes
  per clip slot (distinct tones per clip).
- **D-3 (Lane B, P1):** one-shot `zing setup` can never finish a cold
  onboarding — `h_study_video` daemon threads die with the CLI process
  (<1s), statuses wedge at `running`, and re-runs start nothing. The
  CLI needs to either wait on its own jobs or spawn detached workers.
- **D-4 (Lane B):** `advance_setup` never restarts `failed` studies —
  after F-03 reconcile, `zing setup` parks forever; the reconcile
  message says "call study_video again" but setup never does.
- **D-5 (Lane B/A):** `zing setup --pack` builds the preset profile
  without the pack's manifest provenance (no pack_id/manifest_sha/
  per-ref outcomes) — route it through `build_pack` or stamp
  `preset_pack` equivalently.
- **D-6 (Lane B):** `zing doctor` tts fix-hint promises "zing render
  fetches the kokoro model on first VO use"; tts.py by design never
  downloads — hint lies, fix the wording (and decide who fetches).
- **D-7 (Lane A):** the assemble path (`draft_for_slug`) has no CLI or
  MCP surface — Track 1's direction→draft step is unreachable outside
  Python imports.
- **D-8 (Lane A/C):** no production path derives caption specs from a
  breakdown's measured words — draft EDLs render caption-less without
  hand glue; the S4 "captions from measurements" step needs an owner.
- **D-9 (Lane B, env):** yt-dlp without a JS runtime is now FATAL for
  YouTube (S2's warning came true); doctor should fail-or-warn on
  missing deno AND surface bot-gate errors distinctly. Related: B-Q12
  (yt-dlp not in `[study]` extras) still open.
- **O-1 (Lane A, observation):** the ai-tech-talking-head pack pools
  vertical shorts and horizontal long-forms into one profile — its own
  coherence warning fires and pooled caption stats are dominated by the
  long-forms; consider per-orientation variant packs (A-Q14 wording
  already allows it).
